# E2E Test Task: Overview Dashboard + Release Control

Write two Playwright E2E test files for the adminPortal.

## Context
- Test dir: `apps/adminPortal/tests/`
- Config: `apps/adminPortal/playwright.config.ts` (webServer auto-starts dev frontend on port 3000)
- Helper: `apps/adminPortal/tests/helpers.ts` has `login(page)` function
- All tests need `import { test, expect } from '@playwright/test';` and `import { login } from './helpers';`
- The backend (CoreApi) runs on port 11451 — tests hit real API via the frontend proxy
- Follow existing test patterns in `users.spec.ts`, `tenants.spec.ts`, etc.

## File 1: `apps/adminPortal/tests/overview.spec.ts`

Dashboard overview page (`/overview`) tests. Read `apps/adminPortal/src/components/business/managementOverviewPanel.tsx` to understand the UI.

Cover:
1. Stats cards render (online users, conversations, messages, active tenants, uptime)
2. System health indicators display (database, Redis status)
3. Quick action links work (navigate to /users, /tenants, /audit)
4. Recent activity timeline shows entries or empty state
5. Page refreshes data correctly (reload)
6. Loading state while data fetches

## File 2: `apps/adminPortal/tests/release.spec.ts`

Release control panel (`/release`) tests. Read `apps/adminPortal/src/components/business/releaseControlPanel.tsx` to understand the UI.

Cover:
1. Release checks list renders with status indicators (done/not done)
2. Completion progress bar or percentage displays
3. Loading state while fetching checks
4. Error state with retry button
5. Individual check items display correct titles

## Rules
- Use `test.describe` to group tests
- Use `test.beforeEach` with `login(page)` for authentication
- Navigate with `await page.goto('/overview')` or `await page.goto('/release')`
- Use `page.getByText()`, `page.getByRole()`, `page.locator()` for selectors
- Add `test.slow()` if a test needs more time
- Do NOT mock API calls — tests hit the real backend
- Zero lint errors — run `npm run lint` to verify
