# E2E Test Task: Permission Enforcement + Mobile Layout + 404/Sidebar

Write three Playwright E2E test files for the adminPortal.

## Context
- Test dir: `apps/adminPortal/tests/`
- Config: `apps/adminPortal/playwright.config.ts` (projects: chromium + mobile-chrome/Pixel 5)
- Helper: `apps/adminPortal/tests/helpers.ts` has `login(page)` function
- All tests need `import { test, expect } from '@playwright/test';` and `import { login } from './helpers';`
- The backend (CoreApi) runs on port 11451 — tests hit real API via the frontend proxy
- Follow existing test patterns in `users.spec.ts`, `settings.spec.ts`, etc.

## File 1: `apps/adminPortal/tests/permissions.spec.ts`

Permission/RBAC enforcement tests. Read `apps/adminPortal/src/stores/useAuthStore.ts` and route components to understand permission gating.

Cover:
1. User with viewer role can access overview but NOT settings/release
2. Navigation menu hides items user lacks permission for
3. Direct URL navigation to unauthorized page shows permission denied message or redirects
4. Admin user can access all pages

Note: You may need to create a test user with limited roles via the API, or check how the existing auth flow sets permissions. Read `apps/adminPortal/src/app/appRoot.tsx` for permission checks.

## File 2: `apps/adminPortal/tests/mobile.spec.ts`

Mobile responsive layout tests. Read `apps/adminPortal/src/components/business/messagePanel.tsx` and `apps/adminPortal/src/app/layout/sidebar.tsx` for mobile-specific code.

Use `test.use({ ...devices['Pixel 5'] })` to set mobile viewport.

Cover:
1. Sidebar collapses on mobile viewport, hamburger menu visible
2. Sidebar opens/closes when hamburger is tapped
3. Overview page renders without horizontal overflow on mobile
4. Users table scrolls horizontally on narrow viewport
5. IM conversation panel slides over list on mobile (if navigated to /im)

## File 3: `apps/adminPortal/tests/navigation.spec.ts`

Sidebar navigation + 404 tests. Read `apps/adminPortal/src/app/layout/sidebar.tsx` and `apps/adminPortal/src/app/appRoot.tsx`.

Cover:
1. Sidebar renders all navigation items for admin user
2. Clicking each nav item navigates to correct route
3. Active route is highlighted in sidebar
4. Navigating to invalid URL (e.g. /nonexistent-page) shows 404 page
5. 404 page has a "back home" link that navigates to /overview

## Rules
- Use `test.describe` to group tests
- Use `test.beforeEach` with `login(page)` for authentication
- Navigate with `await page.goto('/path')`
- Use `page.getByText()`, `page.getByRole()`, `page.locator()` for selectors
- Add `test.slow()` if a test needs more time
- Do NOT mock API calls — tests hit the real backend
- Zero lint errors — run `npm run lint` to verify
