# E2E Test Task: IM Chat

Write a Playwright E2E test file for the adminPortal IM chat feature.

## Context
- Test dir: `apps/adminPortal/tests/`
- Config: `apps/adminPortal/playwright.config.ts` (webServer auto-starts dev frontend on port 3000)
- Helper: `apps/adminPortal/tests/helpers.ts` has `login(page)` function
- The backend (CoreApi) runs on port 11451 — tests hit real API via the frontend proxy
- Follow existing test patterns in `users.spec.ts`, `tenants.spec.ts`, etc.
- IM component: `apps/adminPortal/src/components/business/messagePanel.tsx` (~1150 lines)
- IM hooks: `apps/adminPortal/src/hooks/useImSocket.ts` (WebSocket via Socket.IO)

## File: `apps/adminPortal/tests/im.spec.ts`

Cover these scenarios:

1. **Conversation list renders** — navigate to /im, verify conversation list loads
2. **Select conversation** — click a conversation, verify messages display
3. **Send text message** — type in input, click send, verify message appears in list
4. **Receive message** — this may need two browser contexts or be skipped if WebSocket is hard to trigger
5. **Message edit** — click edit on own message, change text, save, verify updated content
6. **Message delete** — click delete on own message, confirm, verify message removed
7. **Typing indicator** — type in input, verify typing indicator shows (or is sent via WS)
8. **Image upload area** — verify the upload/drop zone renders (even if file upload is hard to test without fixtures)
9. **Offline/reconnect indicator** — verify connection status UI exists
10. **Empty state** — what shows when no conversations exist

## Rules
- Use `test.describe` to group tests
- Use `test.beforeEach` with `login(page)` for authentication
- Navigate with `await page.goto('/im')`
- Use `page.getByText()`, `page.getByRole()`, `page.locator()` for selectors
- For WebSocket-dependent tests, if the real WS connection cannot be established in test, use `test.skip()` with a comment explaining why
- Add `test.slow()` if a test needs more time
- Do NOT mock API calls — tests hit the real backend
- Zero lint errors — run `npm run lint` to verify
